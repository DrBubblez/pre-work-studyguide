Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An element refers to to the entire element from open to close, where as
    tag refers only to what appears inside of the angle brackets (< >).
  • An HTML attribute defines additional information about an element.
  • Metadata is data about our webpage that the browser uses to effectively serve users.
    Typically not seen by users, but it contains info about accessibility, search engines,
    and performance.
  • The name of the element describes its content and the role it plays on the webpage.
  • The content is contained between an opening tag and closing tag

CSS

  • Inline CSS is styling an element directly in the HTML file. Inline styles apply to
    a specific HTML tag, using a style attribute with a CSS rule.
  • Internal Style Sheets contain CSS rules for the webpage in the head section of the HTML file.
  • External Style Sheets allow developers to keep all their CSS rules in a separate file.
  • Selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply
    and the value of the property.
  • Margins indicates how much space we want around the outside of an element.
  • Padding indicates how much space we want around the content inside an element.

Git

  • Git is a verson control software, that allows multiple people to make seperate
    changes to a program at the same time.
  • A branch is a workspace where you can make changes that won't affect the
    live site.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • A Repositpory ("repo") is a folder for your project that contains all
    of the files and their verson history.
  • Markdown (.md) is a way to write for the web using plain text; easily
    converts to html.
